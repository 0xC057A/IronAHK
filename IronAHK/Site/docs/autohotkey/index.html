AutoHotkey

Differences

Syntax

The script compiler automatically makes the following conversions to ease transition from AutoHotkey:

  • Set commands such as SetKeyDelay, AutoTrim and DetectHiddenWindows have been replaced with accessors to their respective A_ variables, i.e. SetWorkingDir, %A_Temp% is translated to A_WorkingDir = %A_Temp%;
  • If commands are replaced by the equivalent if-statements, e.g. IfGreater, var, 10 becomes if var > 10. This includes IfMsgBox, IfInString, IfWinActive and others;
  • InStr and SubStr are used in place of older string commands like StringLeft and StringTrimRight;
  • Repeat is converted to Loop;
  • Edit, ListHotkeys, ListLines and ListVars will do nothing until a future version which has improved debugging support.

Arrays

The symbols [ and ] are used as array indexers and can no longer be used in variable names.

Security

When scripts are compiled the source code is not attached to the resulting binary file so it cannot be decompiled. This is unlike AutoHotkey which stores a full copy of the script with the compiled .exe that can be extracted and decrypted or read from process memory when it is executed.

Portability

Scripts will run the same natively on all operating systems and architectures (32/64 bit) with a .NET compatible runtime such as Mono. Almost every version of Windows from XP onwards and most recent Linux desktop distributions have this by default.

Unicode

Source files are read as unicode text. This is the same for all strings and hotkey labels in a script.
